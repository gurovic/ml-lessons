# Формулы в презентации

## Проблема

Изначально формулы были отдельным полем `formula` в JSON и отдельным textbox на слайде. Это ограничивало: одна формула на слайд, фиксированное место, дублирование смысла (текст слева, формула справа).

## Решение

Формулы — **часть обычного текста**. Встречаются в любом количестве в заголовке, буллетах и подписи ссылки. Рендерятся как native Equation (OMML) в PowerPoint, редактируемые двойным кликом.

### Синтаксис в JSON-строках

| Синтаксис | Назначение |
|-----------|------------|
| `$...$` | inline-формула (LaTeX внутри) |
| `$$...$$` | display-формула (отдельная строка в том же абзаце) |
| `\$` | литеральный символ `$` |
| `**...**` | жирный текст (как в markdown; в PowerPoint — `b=1`, без звёздочек) |
| `` `...` `` | inline-код (Consolas, без обратных апострофов в pptx) |

Пример:

```json
{
  "title": "Misclassification Impurity",
  "bullets": [
    "$I_{mis}(t) = 1 - \\max_k p_k$ — вероятность ошибки при мажоритарном ответе",
    "**MSE** — средний квадрат ошибки"
  ]
}
```

В JSON обратный слэш удваивается: `\\sum`, `\\max_k`.

### Отвергнутые альтернативы

| Альтернатива | Почему нет |
|--------------|------------|
| Поле `formula` (отдельный блок) | одна формула, фиксированная вёрстка |
| PNG через matplotlib | не редактируется в PowerPoint |
| Структурированный JSON `[{type: math}, ...]` | громоздко для AI и ручного редактирования |
| Unicode в тексте (`∑`, `log₂`) | не редактируется как Equation |

## Техническая реализация

```
LaTeX в $...$  →  latex2mathml  →  MML2OMML.XSL  →  OMML (a14:m)
```

Модули:

- `agents/omml_converter.py` — LaTeX → OMML
- `agents/rich_text.py` — парсинг `$...$` / `$$...$$`, сборка paragraph (текст + OMML)
- `agents/pptx_builder.py` — все текстовые поля через `rich_text`

OOXML в абзаце:

```xml
<a:p>
  <a:r><a:t>Энтропия: </a:t></a:r>
  <a14:m><m:oMath>...</m:oMath></a14:m>
  <a:r><a:t> — мера неопределённости</a:t></a:r>
</a:p>
```

**Важно:** MathML передаётся в XSLT **с namespace** (`http://www.w3.org/1998/Math/MathML`). Без namespace XSLT возвращает только первый фрагмент формулы.

### Где обрабатывается `$...$`

| Поле | Rich text |
|------|-----------|
| `title` | да |
| `bullets[]` | да |
| `link.label` | да |
| `notes` | нет (plain text) |

### Ошибки конвертации

При сбое LaTeX → OMML: в слайд вставляется сырой текст `$...$`, в лог — предупреждение. Сборка не прерывается.

## Миграция

1. Удалить поле `formula` из JSON-слайдов.
2. Перенести LaTeX в строки с `$...$`.
3. Удалить `agents/formula_renderer.py`.
4. Убрать `add_formula_block` из `omml_converter.py`.

Пилот: слайды `05.json`, `06.json` урока `derevo_resheniy`.
