# Workflow: issue и работа в main

## Режим по умолчанию

**Ветка `main`:** коммиты и push напрямую в `main`. Отдельные feature-ветки и PR — не обязательны.

## Когда создавать issue

| Ситуация | Issue |
|----------|-------|
| Крупная задача (новый урок, новый агент, смена пайплайна) | **Рекомендуется** — чеклист и DoD |
| Архитектурное решение, меняется несколько модулей | **Рекомендуется** |
| Опечатка, однострочный багфикс, правка одного слайда | **Нет** |
| Перегенерация после `author_feedback.md` | **Нет** (достаточно файла и чата) |

Issue — **спецификация для агентов и людей**: цель, чеклист, критерии готовности. Шаблоны: `.github/ISSUE_TEMPLATE/` (`task`, `lesson`).

## Порядок работы

1. **Задача** — в чате, issue (опционально) или `author_feedback.md` / `plan.md`.
2. **Работа** — коммиты в `main`.
3. **Закрытие issue** — вручную или коммитом с `Fixes #N`, если issue был создан.

Опциональный workflow **issue → ветка → PR** допустим для больших изменений, но не является правилом по умолчанию.

## Как используют issue агенты

- Читать issue **до** правок кода — если issue есть.
- **Всегда** читать `lessons/<slug>/author_feedback.md` перед перегенерацией урока (если файл существует).
- Для уроков: содержание слайдов — в **`plan.md`**; issue фиксирует объём работ, если нужен.

## Issue vs plan.md

| | GitHub Issue | `lessons/<name>/plan.md` |
|---|--------------|---------------------------|
| **Область** | Задачи репозитория, пайплайн, агенты | Содержание конкретного урока |
| **Кто пишет** | Автор / ассистент при постановке задачи | Автор урока |
| **Жизненный цикл** | Закрывается вручную или `Fixes #N` | Живёт с уроком |
| **Шаблон** | `.github/ISSUE_TEMPLATE/lesson.md` | Структура слайдов в папке урока |

## author_feedback.md

Постоянный артефакт урока. **Не удалять** при перегенерации: накопленный список прошлых багов и правок pptx. Агенты ориентируются на него при правке JSON, PNG и pptx. См. **docs/pipeline.md**, **RULES.md**.

## Связанные документы

- **RULES.md** — правила для ассистента
- **docs/pipeline.md** — порядок шагов над уроком
