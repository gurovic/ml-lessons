# Мини-проект: project.ipynb

## Проблема

`code.ipynb` даёт короткие фрагменты кода по отдельным слайдам. На занятии не хватает **сквозного сценария**: один датасет, одна задача, все шаги ML-пайплайна от EDA до метрик и диагностики.

## Решение

Дополнительный файл **`project.ipynb`** в папке урока — end-to-end мини-проект на **реальных данных** (sklearn / OpenML, без внешних CSV).

### Отличие от code.ipynb

| | `code.ipynb` | `project.ipynb` |
|---|-------------|-----------------|
| Структура | по заголовкам слайдов | по этапам проекта |
| Данные | синтетика, мини-выборки | один реальный датасет на весь ноутбук |
| Цель | показать API и идею слайда | повторить максимум тем урока в одном сценарии |

### Рекомендуемые этапы

1. **Постановка задачи** — что предсказываем, зачем.
2. **Загрузка и EDA** — таблица, распределения, scatter, корреляции.
3. **Проблемы данных** — выбросы, пропуски (если есть), дисбаланс (классификация).
4. **Разделение** — `train_test_split`, стратификация для классификации.
5. **Pipeline** — препроцессинг + модель без утечки из test.
6. **Базовая модель** — алгоритм урока «из коробки».
7. **Регуляризация / гиперпараметры** — Ridge/Lasso или `C`, `class_weight` и т.п.
8. **Метрики** — метрики из слайдов на hold-out.
9. **Диагностика** — графики остатков / ROC–PR / калибровка.
10. **Интерпретация** — веса, odds, feature importance.
11. **Ограничения** — где модель ломается (нелинейность, XOR).
12. **Чек-лист** — краткое резюме практики.

### Правила

- `random_state=42` везде, где применимо.
- Не ссылаться на номера слайдов; темы урока — по смыслу этапов.
- Графики: подписи осей на русском, `figsize` читаемый, `%matplotlib inline`.
- Один ноутбук ≈ 45–60 мин практики после лекции (не дублировать всю теорию).

### Датасеты по урокам (рекомендации)

| Урок | Датасет | Зачем |
|------|---------|-------|
| Линейная регрессия | `fetch_california_housing` | многомерность, выбросы, масштаб, Ridge/Lasso |
| Логистическая регрессия | `load_breast_cancer` + OpenML `credit-g` | Pipeline, ROC/PR, дисбаланс, калибровка |
| Дерево решений | `load_iris` / `load_wine` | критерии, глубина, feature importance |

## Отвергнутые альтернативы

| Альтернатива | Почему нет |
|--------------|------------|
| Расширить `code.ipynb` одним большим блоком | теряется связь со слайдами; ноутбук раздувается |
| Внешние CSV | лишние зависимости и лицензии; sklearn/OpenML достаточно |
| Авто-генератор как у notebook_generator | сценарий проекта слишком предметный; пока вручную |
