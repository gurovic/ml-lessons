# Агент-рецензент урока

## Проблема

После генерации JSON-слайдов агентами нужна независимая проверка: факты, логика, ясность, лаконичность, полнота. Автор **не** правит JSON вручную — основная QA в `presentation.pptx` после сборки (см. **docs/pipeline.md**). Рецензент по JSON полезен **до** первой сборки pptx или если pptx ещё не собран.

## Решение

Агент **lesson_reviewer** формирует промпт для AI-рецензента. AI возвращает структурированный отчёт в `review.md` (перезаписываемый); **агенты** (не автор) вносят правки в `slides_json/` по замечаниям, затем — визуализации и `pptx_builder`.

Замечания автора **после** проверки `presentation.pptx` — в **`author_feedback.md`** (чеклисты по слайдам). Этот файл **не** читается и **не** перезаписывается `lesson_reviewer`.

### Когда включать

**Опционально после** генерации всех JSON в `slides_json/`, **до** `--visuals` и `pptx_builder`. Можно пропустить и проверять только `presentation.pptx` после сборки.

### Критерии рецензии

| # | Критерий | Что проверять |
|---|----------|---------------|
| 1 | Фактология | Ошибки в определениях, формулах, утверждениях о sklearn/алгоритмах |
| 2 | Логика развития | Связность слайдов внутри урока; согласованность с другими уроками курса |
| 3 | Понятность | Аудитория — продвинутые старшеклассники |
| 4 | Лаконичность | Нет воды, «красивых фраз», расплывчатых формулировок |
| 5 | Полнота | Соответствие `plan.md` |
| 6 | Визуализации | Покрытие графиками/схемами; несколько иллюстраций где уместно (см. `docs/visuals.md`) |

**Отсылки в отчёте:** только по заголовку слайда (`title`), в «ёлочках». Не по номеру файла или порядковому номеру (см. RULES.md).

### Отвергнутые альтернативы

| Альтернатива | Почему нет |
|--------------|------------|
| Автоправка JSON агентом | пользователь должен контролировать правки |
| Рецензия после PPTX | для финальной QA достаточно правок в PowerPoint; JSON-рецензия — ранняя страховка |
| JSON-отчёт | markdown удобнее читать и править |

## Формат отчёта

Файл AI-рецензии: `lessons/<lesson>/review.md` (опционально, перезаписывается при `--save`).

Замечания автора после pptx: `lessons/<lesson>/author_feedback.md` — отдельный persistent-файл, см. **docs/pipeline.md**.

Структура (задаётся промптом):

```markdown
# Рецензия: <тема>

## Резюме
Краткий вердикт: готов / нужны правки / серьёзные проблемы.

## 1. Фактология
### Критичные
- **«Заголовок слайда»** — ...
### Замечания
- ...

## 2. Логика развития темы
### Внутри урока
...
### Связь с другими уроками
...

## 3. Понятность
...

## 4. Лаконичность
...

## 5. Полнота
### Пропуски относительно plan.md
...
### Лишнее
...

## 6. Визуализации
...

## Приоритетные правки
1. ...
```

## Техническая реализация

- `agents/lesson_reviewer.py` — сбор контекста, вывод промпта, `--save`
- `agents/prompts/lesson_reviewer.md` — system prompt

**Вход агента:**
- `plan.md`, `info.json`, все `slides_json/*.json`
- Краткий контекст других уроков: `README.md` + первые строки `plan.md` из `lessons/*/`

**CLI:**

```bash
python agents/lesson_reviewer.py <lesson_dir>           # вывести промпт
python agents/lesson_reviewer.py <lesson_dir> --save  # сохранить отчёт (stdin или файл)
python agents/lesson_reviewer.py <lesson_dir> --save review_response.md
```

## Порядок в пайплайне

См. **docs/pipeline.md**. Кратко:

1. plan.md → генерация JSON (агенты)
2. **Опционально:** lesson_reviewer → review.md → агенты правят JSON
3. --visuals → pptx_builder → **автор проверяет presentation.pptx** → замечания в `author_feedback.md`
4. notebook_generator → code.ipynb → references_agent
