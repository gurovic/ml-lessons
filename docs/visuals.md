# Визуализации на слайдах

## Проблема

Слайды часто остаются без иллюстраций: в JSON поле `visuals` пустое, хотя тема наглядна (график данных, схема алгоритма, сравнение метрик). Одна картинка на весь урок недостаточна; сложные слайды требуют **нескольких** иллюстраций (до 4 на слайд).

## Решение

### Политика «визуализируй всё, что можно»

1. **По умолчанию** у каждого слайда есть хотя бы одна иллюстрация.
2. **Исключения** (можно `visuals: []`):
   - чистый чек-лист / итоги без новых понятий;
   - слайд только со ссылкой на внешний интерактив;
   - определение, которое нельзя показать кроме формулы в тексте (редко).
3. **Несколько иллюстраций** (2–4) — когда на слайде несколько независимых идей:
   - «до/после», «с/без», OLS vs Ridge;
   - схема + график;
   - два разных аспекта одной темы (геометрия штрафа + bar chart весов).
4. Одна картинка может быть **multi-panel** (`subplots`) — тогда в `visuals` один элемент с описанием всех панелей.

### Что визуализировать (чеклист для генератора слайдов)

| Тип темы | Примеры иллюстраций |
|----------|---------------------|
| Данные и модель | scatter + линия/граница, остатки, предсказания vs факт |
| Геометрия / интуиция | L1 vs L2, vertical vs perpendicular distances, decision regions |
| Алгоритм | block-схема (graphviz), дерево, шаги жадного построения |
| Сравнение | bar chart метрик/весов, train vs test, до/после preprocessing |
| Распределения | гистограмма остатков, QQ-plot, классы в 2D |
| Диагностика | leverage/outliers, learning curve, validation curve |
| Предобработка | до/после scaling, one-hot vs label encoding (схема) |

Если буллет утверждает «X влияет на Y» — добавь график или схему, где это видно.

### Педагогический подбор данных

Иллюстрация должна **наглядно доказывать** тезис слайда, а не «формально показывать график». Данные подбирают (хардкод или seed), чтобы эффект был **виден без увеличения** и без подписи «смотрите внимательно».

| Цель | Как подобрать данные |
|------|----------------------|
| До / после | Две панели с **явным** контрастом (наклон линии, разброс, метрика) |
| «X хуже Y» | RMSE заметно больше MAE → один **крупный** промах в выборке |
| Переобучение / train vs test | $R^2$ на train высокий, на test низкий — test с шумом/сдвигом, не случайный split |
| Выброс | Точка **далеко** от облака; линия без неё и с ней **заметно** различаются |
| Leverage | Точка далеко по $x$ при нормальном $y$; наклон меняется |
| Мультиколлинеарность | corr($x_1,x_2$) ≳ 0.95; bootstrap-веса с **широким** разбросом |
| Lasso → 0 | $\alpha$ и признаки так, чтобы 1–2 веса **точно** обнулились |
| «Воронка» остатков | дисперсия остатков растёт с $\hat{y}$ (напр. $y = x + \varepsilon \cdot x^2$) |
| Нелинейность | парабола / синус vs прямая — **U-образный** или изогнутый паттерн |
| Масштабирование | признаки разных порядков (м² vs этаж) → веса **несопоставимы** до scaler |

**Проверка перед сохранением PNG:** «Поймёт ли ученик мысль за 3 секунды, глядя только на картинку?» Если нет — усилить контраст (амplitude, выброс, разница метрик), не добавлять текст.

В `description` в JSON укажи **какой эффект должен быть виден** (не только тип графика): «RMSE заметно выше MAE из-за одного промаха +7».

### Формат JSON

```json
"visuals": [
  {
    "description": "Scatter: точки (x,y), красная линия OLS; подписи осей; seed 42.",
    "output": "ols_scatter.png"
  },
  {
    "description": "Bar chart: MAE и RMSE двух моделей на одной задаче.",
    "output": "metrics_compare.png"
  }
]
```

- `description` — **техническое задание** для генератора кода: тип графика, оси, цвета, что подписать, синтетические данные.
- `output` — уникальное имя в `assets/`, snake_case, `.png`.
- Имена файлов: `{тема}_{что}.png`, без пробелов.

### Качество PNG (для viz_generator)

- **Стиль:** `agents/viz_style.py` — `apply_matplotlib_slide_style()`, `save_slide_figure()`, `style_axes()`. Обязательно подключать в каждом скрипте генерации.
- **Шрифт:** единая шкала в `viz_style.py` — подписи осей `FONT_LABEL` (18 pt), заголовки панелей `FONT_TITLE` (20 pt), деления `FONT_TICK` (14 pt), легенда `FONT_LEGEND` (14 pt). Для annotate и bar labels — `FONT_ANNOT` (14 pt); блоки схем — `FONT_DIAGRAM` (16 pt); ячейки heatmap — `FONT_HEATMAP` (12 pt). Одна панель в правой колонке: `FIGSIZE_SINGLE` (5.5×4″), не широкий холст.
- **Контраст:** только **тёмный текст (`#1a1a1a`) на белом фоне (`#ffffff`)**. Запрещены тёмные темы matplotlib/seaborn, тёмный `axes.facecolor`, прозрачный фон. На heatmap — `heatmap_text_color()` или светлая colormap + контрастная подпись в ячейке.
- `matplotlib`: `figsize` из `viz_style.py` — `FIGSIZE_SINGLE` (5.5×4) для одной панели в правой колонке; `FIGSIZE_DUAL_COL` / `FIGSIZE_TRIPLE_COL` + `subplots(2,1)` / `(3,1)` для multi-panel в колонке; `FIGSIZE_DUAL_ROW` / `FIGSIZE_TRIPLE_ROW` только для полосы под текстом на всю ширину.
- `savefig` через `save_slide_figure()` — RGB, без прозрачности (иначе на тёмном фоне слайда — чёрные полосы).
- Подписи на **русском**, легенда, заголовок панели.
- Данные: `numpy.random.seed(42)` или **подобранные** массивы (см. «Педагогический подбор данных»).
- Для схем: `graphviz` → PNG; узлы с `fillcolor=white`, `fontcolor="#1a1a1a"`.

### Вёрстка в PPTX (`pptx_builder`)

Правая колонка (~4.8″), высота до области ссылки:

| Число картинок | Раскладка |
|----------------|-----------|
| 1 | на всю колонку |
| 2 | столбик |
| 3–4 | сетка 2×2 |

Сохраняется пропорция; картинка вписывается в ячейку.

### Пайплайн

1. Генератор слайдов заполняет `visuals` по правилам выше.
2. Рецензент проверяет **покрытие** (см. `lesson_reviewer.md`).
3. `--visuals` → промпты с `viz_generator.md` + этот документ.
4. `pptx_builder` вставляет все файлы из `visuals`.

### Отвергнутые альтернативы

| Альтернатива | Почему нет |
|--------------|------------|
| Одна иллюстрация на урок | мало для понимания |
| Автогенерация без описания в JSON | нет связи с конкретным буллетом |
| Только формулы вместо графиков | формулы не заменяют интуицию |
