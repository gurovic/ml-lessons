# Визуализации на слайдах

## Проблема

Слайды часто остаются без иллюстраций: в JSON поле `visuals` пустое, хотя тема наглядна (график данных, схема алгоритма, сравнение метрик). Одна картинка на весь урок недостаточна; сложные слайды требуют **нескольких** иллюстраций (до 4 на слайд).

## Решение

### Политика «визуализируй всё, что можно»

1. **По умолчанию** у каждого слайда есть хотя бы одна иллюстрация.
2. **Исключения** (можно `visuals: []`):
   - чистый чек-лист / итоги без новых понятий;
   - слайд только со ссылкой на внешний интерактив;
   - определение, которое нельзя показать кроме формулы в тексте (редко).
3. **Несколько иллюстраций** (2–4) — когда на слайде несколько независимых идей:
   - «до/после», «с/без», OLS vs Ridge;
   - схема + график;
   - два разных аспекта одной темы (геометрия штрафа + bar chart весов).
4. Одна картинка может быть **multi-panel** (`subplots`) — тогда в `visuals` один элемент с описанием всех панелей.

### Что визуализировать (чеклист для генератора слайдов)

| Тип темы | Примеры иллюстраций |
|----------|---------------------|
| Данные и модель | scatter + линия/граница, остатки, предсказания vs факт |
| Геометрия / интуиция | L1 vs L2, vertical vs perpendicular distances, decision regions |
| Алгоритм | block-схема (graphviz), дерево, шаги жадного построения |
| Сравнение | bar chart метрик/весов, train vs test, до/после preprocessing |
| Распределения | гистограмма остатков, QQ-plot, классы в 2D |
| Диагностика | leverage/outliers, learning curve, validation curve |
| Предобработка | до/после scaling, one-hot vs label encoding (схема) |

Если буллет утверждает «X влияет на Y» — добавь график или схему, где это видно.

### Формат JSON

```json
"visuals": [
  {
    "description": "Scatter: точки (x,y), красная линия OLS; подписи осей; seed 42.",
    "output": "ols_scatter.png"
  },
  {
    "description": "Bar chart: MAE и RMSE двух моделей на одной задаче.",
    "output": "metrics_compare.png"
  }
]
```

- `description` — **техническое задание** для генератора кода: тип графика, оси, цвета, что подписать, синтетические данные.
- `output` — уникальное имя в `assets/`, snake_case, `.png`.
- Имена файлов: `{тема}_{что}.png`, без пробелов.

### Качество PNG (для viz_generator)

- `matplotlib`: `figsize` под узкую колонку PPTX (~4.5″) или `subplots(1,2)` для широкой фигуры.
- `savefig(..., dpi=150, facecolor='white', bbox_inches='tight')`; после сохранения — **RGB**, без прозрачности (иначе на тёмном фоне слайда — чёрные полосы).
- Подписи на **русском**, шрифт читаемый, легенда, заголовок панели.
- Данные: `numpy.random.seed(42)` или фиксированные массивы.
- Для схем: `graphviz` → PNG.

### Вёрстка в PPTX (`pptx_builder`)

Правая колонка (~4.8″), высота до области ссылки:

| Число картинок | Раскладка |
|----------------|-----------|
| 1 | на всю колонку |
| 2 | столбик |
| 3–4 | сетка 2×2 |

Сохраняется пропорция; картинка вписывается в ячейку.

### Пайплайн

1. Генератор слайдов заполняет `visuals` по правилам выше.
2. Рецензент проверяет **покрытие** (см. `lesson_reviewer.md`).
3. `--visuals` → промпты с `viz_generator.md` + этот документ.
4. `pptx_builder` вставляет все файлы из `visuals`.

### Отвергнутые альтернативы

| Альтернатива | Почему нет |
|--------------|------------|
| Одна иллюстрация на урок | мало для понимания |
| Автогенерация без описания в JSON | нет связи с конкретным буллетом |
| Только формулы вместо графиков | формулы не заменяют интуицию |
