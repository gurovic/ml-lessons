### План занятия: Визуализации для машинного обучения

**Слайд 1.** Философия визуализации в ML.
*   Визуализация решает три задачи: понимание данных (EDA), диагностика моделей, объяснение результатов бизнесу.
*   Золотое правило: каждый график должен отвечать на конкретный вопрос. Если не знаете, зачем строите — не стройте.
*   Плохая визуализация приводит к неверным решениям. Хорошая визуализация дает инсайт за 3 секунды.
*   Главные библиотеки: matplotlib (низкоуровневая база), seaborn (статистика из коробки), plotly (интерактив), pandas built-in (быстрые графики).

**Слайд 2.** Выбор библиотеки под задачу.
*   matplotlib — полный контроль, кастомные графики, статьи. Основа для всех остальных библиотек.
*   seaborn — 90% EDA задач. Красивые дефолты, статистика одной строкой. Работает поверх matplotlib.
*   plotly — интерактивные дашборды, презентации (zoom, hover, export). Медленнее на больших данных.
*   pandas built-in (df.plot()) — быстрые разведочные графики прямо из DataFrame.
*   Правило: начинайте с seaborn для EDA, переходите на plotly для интерактива, используйте matplotlib для тонкой настройки.

**Слайд 3.** Быстрый старт: pandas built-in визуализация.
*   ML-задача: моментальный обзор данных без импорта дополнительных библиотек.
*   df.plot(kind='hist'), df.plot(kind='box'), df.plot(kind='scatter', x='col1', y='col2').
*   df['col'].value_counts().plot(kind='bar') — частоты категорий.
*   df.plot.hist(alpha=0.5, bins=30) — overlay гистограмм всех числовых столбцов.
*   Ограничение: меньше контроля над стилем, чем в matplotlib/seaborn.
*   Когда использовать: первые 30 секунд знакомства с новым датасетом.

**Слайд 4.** Matplotlib: архитектура Figure и Axes.
*   ML-задача: понимать иерархию объектов для кастомизации графиков.
*   Figure — «холст», Axes — «оси координат» (сам график).
*   fig, ax = plt.subplots() — создание связки.
*   ax.plot(), ax.scatter(), ax.set_xlabel() — методы Axes.
*   plt.tight_layout() — автоматическая подгонка отступов.
*   Понимание этой иерархии критично для создания сложных фигур с несколькими графиками.

**Слайд 5.** Несколько графиков на одной фигуре: subplot.
*   ML-задача: сравнить распределения, показать до/после, создать dashboard.
*   fig, axes = plt.subplots(2, 2, figsize=(12, 10)) — сетка 2x2.
*   axes[0, 0].hist(train['age']); axes[0, 1].hist(test['age']) — сравнение train/test.
*   plt.subplot(2, 3, i) — альтернативный синтаксис.
*   Общий title: fig.suptitle('Comparison').
*   Важно: используйте constrained_layout=True или tight_layout() для избежания перекрытий.

**Слайд 6.** Кастомизация: цвета, стили, аннотации.
*   ML-задача: сделать графики информативными и презентационными.
*   Цвета: color='red', cmap='viridis', alpha=0.6 (прозрачность).
*   Стили линий: linestyle='--', linewidth=2, marker='o'.
*   Аннотации: ax.annotate('Outlier', xy=(x, y), xytext=(x+1, y+1), arrowprops=dict(arrowstyle='->')).
*   Легенды: ax.legend(loc='best', frameon=False).
*   Сетка: ax.grid(True, alpha=0.3).
*   Для презентаций: увеличивайте fontsize через rcParams или параметр fontscale в seaborn.

**Слайд 7.** Сохранение и экспорт графиков.
*   ML-задача: сохранить графики для отчетов, статей, дашбордов.
*   plt.savefig('plot.png', dpi=300, bbox_inches='tight') — высокое разрешение.
*   Векторные форматы: .svg, .pdf — для статей (масштабируются без потери качества).
*   Прозрачный фон: transparent=True.
*   Для дашбордов: plotly сохраняет в HTML с интерактивностью (fig.write_html('plot.html')).
*   Важно: всегда используйте bbox_inches='tight', иначе обрежутся подписи.

**Слайд 8.** Plotly: интерактивные графики одной строкой.
*   ML-задача: создать графики с hover-подсказками, zoom, filter для исследования данных.
*   import plotly.express as px.
*   px.scatter(df, x='area', y='price', color='city', hover_data=['id']) — scatter с подсказками.
*   px.histogram(df, x='income', color='segment', marginal='box') — гистограмма с box plot.
*   px.line(df, x='date', y='sales') — временные ряды с zoom.
*   fig.show() — интерактивное отображение. fig.write_html() — сохранение.
*   Когда использовать: для презентаций, дашбордов, исследования больших данных с drill-down.

**Слайд 9.** Plotly: продвинутые возможности.
*   ML-задача: 3D визуализация, анимация, субплоты.
*   px.scatter_3d(df, x='x', y='y', z='z', color='target') — 3D scatter с вращением.
*   px.scatter(df, x='x', y='y', animation_frame='year') — анимация по времени.
*   make_subplots() — несколько графиков в plotly.
*   Ограничение: тяжелее для >100k точек (используйте datashader или webgl=True).
*   Для дашбордов: интеграция с Dash/Streamlit.

**Слайд 10.** Распределение одной переменной: гистограмма и KDE.
*   ML-задача: понять распределение признака, нужна ли трансформация (логарифмирование), есть ли моды.
*   sns.histplot(df['income'], kde=True) — гистограмма + кривая плотности.
*   Что ищем: Heavy tail (повод применить log1p), мультимодальность (возможно, смешаны разные группы клиентов), выбросы.
*   Параметр bins: мало — скрыты детали, много — шум.
*   Альтернатива: px.histogram(df, x='income', marginal='violin') в plotly.

**Слайд 11.** Быстрый поиск выбросов: Box plot.
*   ML-задача: найти аномалии по IQR, сравнить распределения по группам.
*   sns.boxplot(data=df, x='category', y='price') — медиана, квантили, выбросы (точки за "усами").
*   Что ищем: Точки за усами (кандидаты на удаление/winsorization), разная ширина коробок (гетероскедастичность), смещение медиан (признак значим).
*   Ограничение: не показывает форму распределения (бимодальность скрывается).
*   Интерактив: px.box(df, x='category', y='price', points='all') — hover на каждой точке.

**Слайд 12.** Сравнение распределений: Violin plot.
*   ML-задача: сравнить распределение таргета по категориальным признакам.
*   sns.violinplot(data=df, x='city', y='income') — объединяет box plot и KDE.
*   Классический инсайт: «В Москве средний доход выше, но форма бимодальная — есть две группы населения». Box plot это скрыл бы.
*   Идеален для анализа таргета в регрессии: сразу видно, где предсказывать сложно.
*   Plotly: px.violin(df, x='city', y='income', box=True, points='outliers').

**Слайд 13.** Связь двух непрерывных признаков: Scatter plot + marginal distributions.
*   ML-задача: увидеть линейную/нелинейную связь, кластеры, выбросы в паре признаков.
*   sns.scatterplot(data=df, x='area', y='price', hue='city').
*   Для совместного просмотра с распределениями: sns.jointplot(kind='hex') — hexbin для больших данных.
*   Что ищем: Линейный тренд, нелинейность (нужны полиномы или деревья), кластеры.
*   Для больших данных (>100k): используйте alpha=0.1 или px.scatter с render_mode='webgl'.

**Слайд 14.** Пропуски в данных: missingno.
*   ML-задача: увидеть паттерны пропусков (случайные или систематические).
*   Библиотека missingno визуализирует пропуски одной командой.
*   msno.matrix(df) — матрица пропусков. msno.dendrogram(df) — кластеризация признаков по паттернам пропусков.
*   Критический инсайт: если пропуски в признаке A коррелируют с пропусками в B — пропуски не случайны (MNAR). Простая импутация средним сломает модель.

**Слайд 15.** Мультиколлинеарность: Heatmap корреляций.
*   ML-задача: найти дублирующие признаки и их связь с таргетом.
*   sns.heatmap(df.corr(), annot=True, cmap='coolwarm', center=0) — цветовая карта с центром в нуле.
*   Что ищем: Признаки с корреляцией >0.9 (кандидаты на удаление), признаки с высокой корреляцией с таргетом.
*   Важно: Пирсон по умолчанию (линейная связь). Для нелинейной используйте Спирмена (method='spearman').
*   Plotly: px.imshow(df.corr(), text_auto='.2f', color_continuous_scale='RdBu_r').

**Слайд 16.** Детектирование дрейфа данных: ECDF и overlay.
*   ML-задача: сравнить распределения train/test/production для выявления Data Drift.
*   ECDF (Empirical CDF): sns.ecdfplot(). Доля объектов меньше X. Не зависит от бинов.
*   Overlay KDE: наложение кривых плотности train и prod.
*   Интерпретация: расхождение кривых означает дрейф, модель может деградировать.
*   Plotly: px.ecdf(train_df, x='age', color='dataset') + overlay test_df.

**Слайд 17.** Дисбаланс классов: Count plot.
*   ML-задача: увидеть распределение классов в классификации.
*   sns.countplot(data=df, x='target') — частоты каждого класса.
*   Критический инсайт: если соотношение 100:1 — accuracy бесполезна, нужны class_weight, PR-AUC.
*   Всегда стройте в начале проекта по классификации.
*   Альтернатива: df['target'].value_counts().plot(kind='bar') или px.histogram(df, x='target').

**Слайд 18.** Границы решений: Decision Boundaries.
*   ML-задача: визуализировать, как классификатор разделяет пространство признаков.
*   DecisionBoundaryDisplay.from_estimator(model, X, y) — для 2D классификаторов.
*   Что видим: Линейная граница (логистическая регрессия), нелинейная граница (деревья, SVM RBF), переобучение.
*   Идеально для преподавания: наглядно показывает, почему XOR не решается линейной моделью.
*   Кастомизация: alpha=0.3 для прозрачности, overlay scatter точек.

**Слайд 19.** Многомерные данные: PCA/t-SNE/UMAP.
*   ML-задача: визуализировать многомерные данные в 2D/3D, увидеть кластеры.
*   PCA — линейное снижение размерности, сохраняет глобальную структуру.
*   t-SNE и UMAP — нелинейные методы, сохраняют локальные кластеры. UMAP быстрее.
*   Важно: t-SNE/UMAP только для визуализации, не для создания признаков модели.
*   Plotly 3D: px.scatter_3d(x=pca[:,0], y=pca[:,1], z=pca[:,2], color='target').

**Слайд 20.** Кластеризация: Silhouette plot и Elbow method.
*   ML-задача: выбрать оптимальное число кластеров K в K-means.
*   Elbow method: график WCSS от K. Ищем «локоть».
*   Silhouette plot: показывает качество кластеризации для каждого объекта. Значения близкие к 1 — хорошая кластеризация.
*   Визуализация silhouette scores по кластерам сразу показывает проблемные группы.
*   fig, (ax1, ax2) = plt.subplots(1, 2) — elbow и silhouette рядом.

**Слайд 21.** Диагностика регрессии: Residual plot.
*   ML-задача: проверить качество регрессии, найти систематические ошибки.
*   sns.residplot(x=y_pred, y=y_true - y_pred, lowess=True) — остатки vs предсказания.
*   Хорошая модель: хаотичное облако вокруг нуля (гомоскедастичность).
*   Плохие паттерны: Парабола (не учтена нелинейность), воронка (гетероскедастичность), сдвиг от нуля.
*   Plotly: px.scatter(x=y_pred, y=residuals, trendline='lowess').

**Слайд 22.** Нормальность остатков: Q-Q plot.
*   ML-задача: проверить нормальность ошибок (важно для доверительных интервалов).
*   statsmodels.api.qqplot(residuals, line='s') — квантили остатков vs квантили нормального распределения.
*   Интерпретация: точки должны лежать на прямой. Отклонения на хвостах означают тяжелые хвосты.
*   Для ML: если хвосты тяжелые, используйте MAE/Huber вместо MSE.

**Слайд 23.** Предсказание vs Реальность.
*   ML-задача: визуально оценить качество регрессии.
*   sns.scatterplot(x=y_true, y=y_pred) + диагональная линия y=x.
*   Хорошая модель: точки плотно вокруг диагонали.
*   Добавление R2 и RMSE в заголовок: ax.set_title(f'R2={r2:.2f}, RMSE={rmse:.2f}').
*   Plotly: px.scatter(x=y_true, y=y_pred, trendline='ols') + диагональ.

**Слайд 24.** Диагностика классификации: Confusion Matrix.
*   ML-задача: понять, какие классы модель путает.
*   ConfusionMatrixDisplay.from_estimator(model, X, y) — тепловая карта ошибок.
*   Что ищем: Большие off-diagonal элементы, дисбаланс FN vs FP.
*   Нормализация normalize='true' показывает recall для каждого класса.
*   Кастомизация: cmap='Blues', annot=True, fmt='.0f'.

**Слайд 25.** Качество при разных порогах: ROC и PR кривые.
*   ML-задача: выбрать оптимальный порог, сравнить модели.
*   RocCurveDisplay и PrecisionRecallDisplay из sklearn.
*   PR-кривая важнее при сильном дисбалансе классов.
*   Overlay: несколько моделей на одном графике сразу показывает лучшую.
*   Plotly: px.line(x=fpr, y=tpr) + диагональ для случайного классификатора.

**Слайд 26.** Переобучение vs Недообучение: Learning Curve.
*   ML-задача: понять, нужно ли больше данных или более сложная модель.
*   LearningCurveDisplay.from_estimator(model, X, y) — кривые обучения на train/validation.
*   Интерпретация: Train высоко, Val низко — переобучение. Обе низкие — недообучение. Обе высокие — оптимум.
*   Добавление доверительных интервалов: plt.fill_between для визуализации variance.

**Слайд 27.** Кросс-валидация: стабильность модели.
*   ML-задача: оценить variance модели, а не только среднее качество.
*   Box plot метрик по фолдам: sns.boxplot(data=cv_scores).
*   Line plot: plt.plot(cv_scores, marker='o') — видны аномальные фолды.
*   Интерпретация: большой разброс означает нестабильность модели.
*   Сравнение моделей: sns.boxplot(data=[cv_scores_model1, cv_scores_model2]).

**Слайд 28.** Подбор гиперпараметров: Grid Search visualization.
*   ML-задача: выбрать лучшие гиперпараметры, увидеть взаимодействия.
*   Heatmap: sns.heatmap(grid_results) для 2D grid search (max_depth vs min_samples_leaf).
*   Line plot: зависимость метрики от одного параметра с доверительными интервалами.
*   Инсайт: часто есть «плато» — широкий диапазон параметров дает схожее качество.
*   Plotly: px.imshow для интерактивного heatmap с hover.

**Слайд 29.** Важность признаков: Permutation Importance.
*   ML-задача: понять, какие признаки влияют на предсказания.
*   Для деревьев: model.feature_importances_. Смещено в сторону high-cardinality признаков!
*   Permutation Importance (честнее): перемешиваем значения признака, смотрим падение метрики.
*   sns.barplot(x=result.importances_mean, y=feature_names).
*   Визуализация с error bars: plt.barh(y, x, xerr=std) для показа variance важности.

**Слайд 30.** Стили и темы: единый дизайн для проекта.
*   ML-задача: создать консистентный визуальный стиль для всех графиков проекта.
*   plt.style.use('seaborn-v0_8-whitegrid') — готовые стили matplotlib.
*   sns.set_theme(style='whitegrid', palette='husl') — тема seaborn.
*   Кастомная палитра: sns.set_palette(['#FF6B6B', '#4ECDC4', '#45B7D1']).
*   rcParams: plt.rcParams['figure.figsize'] = (12, 8), plt.rcParams['font.size'] = 12.
*   Для презентаций: увеличенный шрифт, минималистичный стиль, контрастные цвета.

**Слайд 31.** Общие техники: overlay и комбинирование графиков.
*   ML-задача: показать несколько распределений/моделей на одном графике для сравнения.
*   Overlay гистограмм: plt.hist(train, alpha=0.5, label='train'); plt.hist(test, alpha=0.5, label='test').
*   Overlay линий: plt.plot(train_scores); plt.plot(val_scores).
*   Двойные оси (осторожно!): ax2 = ax1.twinx() — для разных масштабов, но часто вводит в заблуждение.
*   FacetGrid в seaborn: g = sns.FacetGrid(df, col='category'); g.map(sns.histplot, 'value').
*   Plotly subplots: make_subplots(rows=2, cols=2) — несколько интерактивных графиков.

**Слайд 32.** Размер и DPI: подготовка для разных целей.
*   ML-задача: оптимизировать размер и качество графика для статьи, презентации, дашборда.
*   figsize=(width, height) в дюймах: (10, 6) для статьи, (16, 9) для презентации.
*   DPI (dots per inch): 100 для экрана, 300 для печати, 600 для журналов.
*   plt.savefig('plot.png', dpi=300, bbox_inches='tight', facecolor='white').
*   Для веба: оптимизация размера файла через формат (SVG < PNG для векторной графики).
*   Aspect ratio: золотое сечение 1.618:1 часто выглядит эстетично.

**Слайд 33.** Антипаттерны и best practices.
*   Антипаттерны: Pie charts (замените на bar), 3D графики (искажают), rainbow colormaps (не colorblind-friendly), отсутствие labels.
*   Best practices: один график = один инсайт, всегда добавляйте title/labels/units, тестируйте на коллегах (5 секунд на понимание).
*   Colorblind-friendly: используйте viridis, plasma, избегайте красно-зеленых сочетаний.
*   Для презентаций: минимум текста, крупные шрифты, контрастные цвета, белый фон.
*   Для статей: векторные форматы (SVG/PDF), grayscale-friendly палитры.

**Слайд 34.** Итоговый чек-лист: какой график для какой задачи.
*   Распределение -> hist + KDE (seaborn/plotly) | Выбросы -> box plot
*   Сравнение групп -> violin plot | Связь признаков -> scatter + jointplot
*   Пропуски -> missingno | Мультиколлинеарность -> heatmap
*   Дрейф -> ECDF, overlay KDE | Дисбаланс -> count plot
*   Границы решений -> DecisionBoundaryDisplay | Многомерные -> PCA/t-SNE
*   Качество регрессии -> residual plot, Q-Q, predicted vs actual
*   Качество классификации -> confusion matrix, ROC/PR curves
*   Переобучение -> learning curve | Важность признаков -> permutation importance
*   Библиотеки: seaborn для EDA, plotly для интерактива, matplotlib для кастомизации, pandas для быстрых графиков.
